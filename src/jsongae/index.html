Talking JSON-RPC 2.0 with Google App Engine

Remote Procedure Calls The Other Way

Remote Procedure Calls are commonly synchronous. This means that a client expects an immediate response with some kind of result from the server.

In contrast, JSON-RPC allows for bidirectional, asynchronous communication between the service and the client, treating each more like peers and allowing peers to call one another or send notifications to one another.

Think of a message board. The client receives a Remote Procedure Call from the server to store an incoming message client-side. This could happen even when the board isn't shown, for instance, when the user composes a new message. Especially mobile web applications don't have much space to show a message board and an input box at the same time.

It also allows multiple commands to be sent to a peer at once, which can be highly useful to keep network traffic low. Bear in mind that these messages may be answered out of order.
